# 变更记录：运行健壮性与热重载陷阱

> 日期：2026-06-19
> 文件：`streamlit_app.py`、`app/llm.py`
> 范围：前端交互修复 + LLM 调用健壮性　|　不改后端业务逻辑

---

## 1. 三个现象

1. **点击示例按钮没有任何输出**（仍停在空状态）。
2. 统计类问题答案其实正确（"知识库里有 24 个文档"），但 **Faithfulness 显示 0.00 / 证据支撑不足**。
3. 选"知识库统计"时，**好几次转圈之后什么都不显示**。

---

## 2. 现象 1：示例点击不运行

### 之前为什么
输入框用 `st.text_input("...", value=default_q)`，示例按钮把问题写进 `pending_q`，再 `pop` 给 `default_q`。
**坑**：Streamlit 对**无 key** 的 `text_input`，在"按钮赋值"场景下经常**不回显** `value=` → `query` 仍为空 →
运行门 `(go or default_q) and query.strip()` 因 `query.strip()` 为假而不触发 → 空状态。

### 现在为什么这么改
改用 **`key` 绑定 `session_state`** 的标准写法：
- `text_input(..., key="qbox")`，按钮（主区示例 + 侧边栏"试一试"）直接写 `st.session_state["qbox"]` 并置 `_autorun=True`；
- 运行门改为 `if (go or _autorun) and query.strip()`。
按钮在输入框实例化**之前**写入 `qbox`（允许），点击当次即回显并自动运行一次，彻底不依赖 `value=` 回显。

---

## 3. 现象 2：答案对、Faithfulness 却 0.00（热重载陷阱）

**答案是对的**；0.00 的根因是：**运行中的进程还在用旧的 `judge.py`**。

### 机制（值得记住）
**Streamlit 的热重载只重跑主脚本 `streamlit_app.py`，不会重新 import 子模块**
（`app/graph/judge.py`、`nodes.py`、`builtins.py` 缓存在 `sys.modules`）。所以：
- "示例点击"修复在主脚本里 → 立即生效；
- "critic 纳入工具结果"修复在 `judge.py` 子模块里 → **未加载**。

旧 judge 只看 5 个检索 chunk（不含 "24"）→ 正确判"24 不被检索证据支撑"→ 0.00。
**解决**：**完全重启** Streamlit（终端 Ctrl+C → 重新 `streamlit run`），让子模块修复生效。

> 经验：改了 `app/**` 下的子模块，必须重启进程；只改主脚本才可靠热重载。

---

## 4. 现象 3：转圈不显示（同源放大 + 接口无超时）

### 根因
1. **同源放大**：旧 judge 把 kb_stats 误判不可信 → critic 重试满 3 轮 →
   一次统计查询要打约 **7 次** LLM（改写 1 + 3×(写作+裁判)）。
2. **致命点**：`OpenAI()` 客户端**没设超时**，某次调用挂起就**无限转圈**；
   7B 第三方端点偶发慢/超时，调用越多越容易中招——所以"知识库统计"最容易卡。

### 现在为什么这么改
- `app/llm.py`：两处 `OpenAI()` 都加 `timeout=30.0, max_retries=1` → 杜绝无限挂起，超时即报错。
- `streamlit_app.py`：`run_query` 包 `try/except` → 出错显示红字提示并 `st.stop()`，不再空白/无限转圈。
- 配合现象 2 的重启：judge 修复生效 → kb_stats 1 轮结束 → 调用数从 ~7 降到 ~3，更稳。

---

## 5. 验证

- 示例点击：改为 key 绑定后，点击当次即填入并自动运行（Read 核对：`qbox`/`_autorun` 各 4 处、无 `default_q/pending_q` 残留）。
- 超时/兜底：Read 核对两处 client 已带 `timeout=30.0, max_retries=1`；run_query 已包 try/except。
- 注：bash 沙箱对 `streamlit_app.py` 的挂载视图有同步延迟，未用其做编译校验；真实文件以文件工具/Windows 路径为准，已逐段 Read 核对。

## 6. 风险与回退

- **风险低**：纯前端交互 + 网络调用参数；后端 graph/检索/工具/judge 业务逻辑未改。
- **回退**：去掉 `timeout/max_retries` 即恢复旧网络行为；运行门可还原为 `default_q` 方案；`try/except` 可移除。

## 7. 操作清单（用户侧）

1. 终端 `Ctrl+C` 完全停掉，再 `streamlit run streamlit_app.py`（让子模块修复全部加载）。
2. 上方把 Chat 模型切到 32B+（弱 7B 当裁判仍可能吐坏 JSON 回退启发式）。
3. 点"知识库统计"：应显示 24 且 faithful、不再无限转圈；端点真超时会显示红字提示，可换模型重试。
