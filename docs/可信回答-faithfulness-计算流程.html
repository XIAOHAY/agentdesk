可信回答（faithfulness）是怎么算出来的

面板里的环形百分比与「可信回答 / 证据支撑不足」的判定流程。逻辑取自 app/graph/judge.py 与 streamlit_app.py。

启发式兜底的公式

score = |分词(答案) ∩ 分词(证据 ∪ 工具结果)| / |分词(答案)|

弱模型吐不出可解析 JSON 时退到这条。纯数字短答案天生偏低（虚词拉低分母），所以真正稳的是 LLM 裁判路径。

走一遍："知识库里有多少个文档？"

答案"知识库里有 24 个文档"是对的。走的是 LLM 裁判（method=llm）。

  • 旧代码：只把 5 个检索 chunk 当证据，里面没有"24" → 模型判"24 无支撑" → score≈0 → < 0.6 → ⚠️ 证据支撑不足 → 环形 round(0×100)=0%、红色。
  • 修复后：judge() 多收"工具结果 = 文档总数=24"并声明为权威依据 → 模型看到 24 就在工具结果里 → 判 faithful → 绿色。

代码位置

  • 分支选择与两种打分：app/graph/judge.py 的 judge() / _heuristic()
  • 阈值：THRESHOLD = 0.6（judge.py）
  • 环形百分比与红/黄/绿取色：streamlit_app.py（pct = round(score*100)、gc 按 0.4 / 0.6 分档）
